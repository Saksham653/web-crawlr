What it extracts

Six data types,
one URL

No configuration. No selectors to write. Crawlr automatically detects and structures every data type on any page.

Headings
Every h1 through h6 element — extracted, tagged with level badges, and fully searchable. H1 displayed in gold, H2 in teal, H3–H6 in muted tones.
h1 → h6
Paragraphs
All <p> text blocks over 40 characters — deduplicated and trimmed. The actual content of the page, not boilerplate. Searchable in real time.
full text content
Links
Every href resolved to an absolute URL. Internal vs external links are automatically flagged. Clickable cards open the URL directly in a new tab.
internal · external
Images
Every <img> src with alt text. Handles lazy-loaded and data-src attributes. Renders as a thumbnail grid with source path and alt text visible.
src · alt · lazy-load
Prices
Regex-matched price amounts with ₹ $ £ € ¥ symbols — anywhere on the page. Each price is parsed to a numeric value for tracker comparisons.
₹ $ £ € ¥ · numeric parse
Meta Tags
All <meta> name/property + content pairs — Open Graph, Twitter Cards, description, keywords, charset. Presented in a clean two-column table.
og · twitter · seo meta
How it works

Simple by design,
powerful by default

01
Paste any URL and hit Scrape
Type a full URL or just a domain — Crawlr adds https:// automatically. Press Enter or click the button. Any publicly accessible webpage works.
02
Auto proxy fallback chain
Three CORS proxies are tried in sequence — allorigins.win → corsproxy.io → codetabs.com. Each has a 10s timeout. The first one that responds wins. Usually takes 2–5 seconds.
03
Browse results across 8 tabs
Overview, Headings, Text, Links, Images, Prices, Meta, Tracker. Every tab has a live search bar. Stats strip shows counts at a glance.
04
Or run Python locally for full power
Sites that block browser proxies work fine locally. The Python scraper uses stdlib urllib with spoofed headers. The dashboard auto-loads the result.
python scraper.py https://any-url.com

Tech stack

Deliberately minimal

No frameworks. No npm. No build step. Just clean, portable code that runs anywhere.

Frontend
Vanilla HTML + CSS + JavaScript. Zero npm packages, zero frameworks, zero build step. Syne · Lora · Fira Code from Google Fonts.
Browser
Python Scraper
Python 3.8+ standard library only — urllib, html.parser, json, re. Zero pip installs. Spoofed User-Agent headers included.
Python 3.8+
CORS Proxy Chain
Three proxies tried in sequence — allorigins.win, corsproxy.io, codetabs.com. Each with a 10s AbortSignal timeout. Auto-fallback on failure.
3 Proxies
Data Storage
localStorage for the price tracker — persists across browser sessions. JSON files on disk for scrape output. No servers, no accounts.
Local Only
Scrapy-Inspired
Architecture mirrors the Scrapy 2.14 framework philosophy — spiders, items, pipelines, and settings — without the dependency overhead.
Scrapy 2.14
Export Engine
JSON · CSV · Markdown · HTML — four formats all built with native browser APIs. DOMParser for parsing, Blob + URL.createObjectURL for downloads.
4 Formats
Honest limitations

Where it shines
and where it doesn't

Works great on
  • Wikipedia — full articles, all sections, all links
  • GitHub — repo pages, README content, file listings
  • Hacker News — posts, comment links, titles
  • News sites — BBC, Reuters, The Hindu, NDTV
  • Blogs & docs — any server-rendered HTML
  • books.toscrape.com — the classic scraping practice site
  • E-commerce with static HTML — prices, images, descriptions
Won't work on
  • Flipkart / Amazon India — Cloudflare blocks all proxies
  • React / Vue / Angular SPAs — JS-rendered, HTML shell is empty
  • Login-gated pages — needs an authenticated session
  • PDFs & image files — no document or OCR parsing
  • Paywalled content — content hidden behind auth
  • WebSocket / SSE pages — real-time data not captured
  • Cloudflare-protected sites — requires headless browser
