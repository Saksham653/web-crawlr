// Documentation & Guide
Using Crawlr
Everything you need to know — from scraping your first page to exporting clean structured data and monitoring prices over time. Read through once and you'll know the whole tool.
Web Scraper Price Tracker 4 Export Formats No Dependencies
01 — Quick Start
Scrape Any Page in 3 Steps
1
Paste any URL into the bar at the top
Type the full URL or just the domain — Crawlr adds https:// automatically. Works with any publicly accessible webpage. Examples: wikipedia.org or https://github.com/torvalds/linux
2
Click Scrape ↵ or press Enter
Crawlr automatically tries three CORS proxies in order — allorigins.win → corsproxy.io → codetabs.com. The status bar shows which proxy is active. Usually completes in 2–6 seconds.
3
Browse the results across 7 tabs
Results appear across Overview, Headings, Text, Links, Images, Prices, Meta, and Tracker tabs. Every tab has a live search bar. Click any link card to open it. The stats bar at the top shows counts at a glance.
💡
Best results on: Wikipedia, GitHub repos, Hacker News, news articles, documentation pages, and any site with standard server-rendered HTML. The more structured the HTML, the richer the extraction.
02 — Extraction
What Gets Pulled from Every Page
Content
  • Headings — all h1–h6, each badged with level
  • Paragraphs — all <p> text over 40 characters
  • Images — src URL + alt text, with thumbnail
  • Page title — from <title> or og:title
Structured Data
  • Links — every href, internal vs external flagged
  • Prices — regex-matched ₹ $ £ € ¥ amounts
  • Meta tags — all name/property + content pairs
  • Open Graph — og:title, og:image, og:description
💡
Prices are parsed with a numeric value too, not just raw text — so ₹1,29,900 becomes both a display string and the number 129900, which is what the price tracker compares against your target.
03 — Price Tracker
Monitor Any Product Price Over Time
1
Scrape the product page first
After scraping, if prices are detected, quick-tap buttons appear at the top of the Tracker tab. Clicking one auto-fills the form with the URL, page title, and selected price as the target. Or fill in manually.
2
Set a target price and click + Add Item
The item is saved in your browser's localStorage. It persists across sessions — even closing the browser tab won't erase it. You can track up to dozens of products simultaneously.
3
Click ↻ to refresh any item
Crawlr fetches the live page and extracts the latest price. The sparkline bar chart at the bottom of each row shows price history — green bars = lowest recorded price, red = highest. The delta badge shows the change vs last reading: ▼ dropped or ▲ went up.
4
Watch for the TARGET HIT badge
When the current price is at or below your target, a ✓ TARGET HIT badge appears on that item. Use Refresh All to bulk-update every tracked product at once.
⚠
Limitation: Price refreshing uses the same browser proxy chain. Sites with heavy anti-bot protection (Flipkart, Amazon India, Myntra) will block live refreshes in the browser. Use the Python scraper locally for those sites — it bypasses most restrictions.
04 — Data Export
Download Your Data in 4 Clean Formats
Click any export button in the toolbar (JSON / CSV / Markdown / HTML Report), choose which sections to include with the checkboxes, preview the output, and hit Download. Files are named automatically: crawlr-wikipedia.org-2026-03-05.json
JSON — Full structured data
  • Nested objects with all fields intact
  • Numeric price values alongside raw strings
  • Includes export timestamp and author credit
  • Best for: developers, APIs, data pipelines
CSV — Spreadsheet-ready
  • Flat rows, one entry per line
  • Opens directly in Excel or Google Sheets
  • Sections separated by blank rows with headers
  • Best for: analysis, filtering, sorting
Markdown — Readable report
  • Clean prose format with proper heading levels
  • Renders in GitHub, Notion, Obsidian, VS Code
  • Links formatted as [text](url)
  • Best for: notes, documentation, sharing
HTML Report — Standalone page
  • Complete dark-themed self-contained webpage
  • All links are clickable, opens in any browser
  • No internet connection needed to view
  • Best for: archiving, sharing with non-technical people
💡
Pro tip: Uncheck sections you don't need before exporting. Deselecting Images keeps file sizes tiny. Deselecting Links gives you a clean text-only report. The preview box updates live so you know exactly what you're downloading.
05 — Python Mode
Run Locally for Unrestricted Scraping
The browser scraper is constrained by CORS — many sites reject requests from third-party proxies. The Python scraper inside the ZIP has none of these limits. It sends real browser-like headers, handles encoding, follows redirects, and writes a clean output/result.json that this dashboard auto-loads on startup.
1
Unzip and open a terminal in the project folder
Requires Python 3.8 or newer. No pip installs needed — only Python standard library is used.
2
Run the scraper with any URL
python scraper.py https://news.ycombinator.com # Custom output path python scraper.py https://example.com -o output/result.json
3
Open this dashboard — data loads automatically
The dashboard checks for output/result.json on every page load. If found, all tabs populate automatically. No need to paste a URL. Re-run the scraper and refresh the page to update.
⚠
Hard wall: Even Python cannot bypass Cloudflare Bot Protection on sites like Flipkart, Amazon India, or Swiggy. Those sites require a headless browser (Selenium / Playwright) — which is beyond the scope of this tool.
06 — Limitations
Know Before You Scrape
Won't work on
  • Login-protected pages — needs a session
  • Flipkart / Amazon India — Cloudflare blocks all proxies
  • React / Vue SPAs — JS renders empty HTML shell
  • PDF / image files — no document or OCR parsing
  • Paywalled articles — content hidden behind auth
Works great on
  • Wikipedia — full article, all sections
  • GitHub — repo pages, README, file listings
  • Hacker News — posts, links, titles
  • News sites — BBC, Reuters, The Hindu, NDTV
  • Blogs & docs — any server-rendered HTML
